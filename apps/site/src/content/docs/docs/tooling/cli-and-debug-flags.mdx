## Build modes

Build the interpreter with the repo-local helper:

```bash
./build.sh
```

Useful variants already supported by the script:

```bash
./build.sh --debug
./build.sh --debug --asan
./build.sh --debug --ubsan
./build.sh --debug --asan --ubsan
./build.sh --release --profiling
./build.sh --release --pgo-generate
./build.sh --release --pgo-use=/tmp/pgo-data
```

Sanitizers cannot be combined with `--lto`, `--pgo-generate`, or `--pgo-use`.

## Running code

```bash
./build/interpreter path/to/program.mog
```

Add extra package roots when needed:

```bash
./build/interpreter --package-path /path/to/packages app.mog
```

## Runtime and diagnostics flags

The current interpreter exposes these runtime/debugging switches:

- `--trace`
- `--show-return`
- `--disassemble`
- `--frontend-timings`
- `--frontend-timings-json`
- `--target <triple>`
- `--prefer-prebuilt`
- `--no-native-build`
- `--cmake-toolchain <path>`

Examples:

```bash
./build/interpreter --trace tests/sample_var.mog
./build/interpreter --show-return tests/sample_var.mog
./build/interpreter --disassemble tests/sample_var.mog
./build/interpreter --frontend-timings tests/sample_var.mog
./build/interpreter --frontend-timings-json tests/sample_var.mog
./build/interpreter run --target linux-arm64-gnu --cmake-toolchain ./toolchains/linux-arm64.cmake app.mog
```

The shell test suite verifies that these flags still execute the target program
while emitting their extra diagnostics.

For package-managed native dependencies, `mog install`, `mog update`, and
`mog run` also accept:

- `--target <triple>` to pin the selected native artifact target
- `--prefer-prebuilt` to prefer registry-published prebuilt artifacts
- `--no-native-build` to reject source-build fallback
- `--cmake-toolchain <path>` to enable non-host native source-build fallback

## Frontend timings

`--frontend-timings` reports a readable phase summary that includes parse,
symbol collection, import resolution, bind, type-check, HIR lowering, HIR
optimization, cache stats, and total frontend time.

`--frontend-timings-json` emits the same information as JSON.

## Package validation

The interpreter also validates package directories:

```bash
./build/interpreter --validate-package packages/examples/math
./build/interpreter --validate-package=packages/examples/counter
```

## Intentional non-feature

The repo explicitly tests that `--strict` is rejected as an unknown option. Mog
is already strictly typed by default; there is no separate runtime strictness
flag today.
