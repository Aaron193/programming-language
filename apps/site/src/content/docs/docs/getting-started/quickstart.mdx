This walkthrough gets you from a built interpreter to a real `.mog` file with
functions, locals, and output.

## First Program

Create a file named `hello.mog`:

```rust
fn applyTwice(f fn(i32) i32, value i32) i32 {
  return f(f(value))
}

var addOne = fn(x i32) => x + 1

print(applyTwice(addOne, 40))
```

Run it:

```bash
./build/interpreter hello.mog
```

## What This Shows

- Named function declarations
- Function types in parameter position
- Local type inference with `var`
- Expression-bodied lambdas
- `print(...)` output through the runtime

## Next Sample

Mog already supports structs, methods, and inferred locals:

```rust
type Counter struct {
  value i32

  fn reset() {
    this.value = 0
  }
}

fn buildCounter() Counter {
  var counter = Counter()
  counter.reset()
  return counter
}

var current = buildCounter()
print(current.value)
```
