The repo’s tooling story is larger than one page now. Use this page as a map of
the deeper tooling guides.

## Read by task

- [CLI and Debug Flags](/docs/tooling/cli-and-debug-flags/) for build modes,
  interpreter entrypoints, diagnostics flags, and package validation.
- [REPL](/docs/tooling/repl/) for interactive use and current limitations.
- [VS Code and LSP](/docs/tooling/vscode-and-lsp/) for the editor extension,
  `mog-lsp`, formatting, hover, navigation, completion, and semantic tokens.

## Current surface area

- interpreter build modes through `build.sh`
- runtime tracing and disassembly flags
- frontend timing output in human and JSON form
- interactive REPL
- VS Code language support
- in-repo LSP with real regression coverage
