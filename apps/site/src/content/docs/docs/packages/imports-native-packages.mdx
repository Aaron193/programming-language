Mog uses `@import(...)` for both source modules and native packages. The package
story is real today, but it is still grounded in the implementation in this
repository.

## Read by task

- [Using Native Packages](/docs/packages/using-native-packages/) for package
  imports, search paths, opaque handles, and validation.
- [Authoring Native Packages](/docs/packages/authoring-native-packages/) for
  manifests, `package.api.mog`, and reference package layout.
- [Window Package](/docs/packages/window-package/) for the first official
  runtime-maintained package.

## Import model at a glance

- string literal paths such as `"./math.mog"` resolve as source modules
- bare names such as `"math"` and `"window"` resolve as packages
- package metadata already exists in `mog.toml`, `mog.lock`, `package.toml`,
  and `package.api.mog`

## Reference packages in this repo

- `packages/mog/window/`
- `packages/examples/math/`
- `packages/examples/counter/`

Treat the current docs as documentation of implemented behavior, not as a
promise that the broader package ecosystem is fully stabilized.
