`window` is the first official runtime-maintained package in the repo. It is
SDL-backed and only builds when SDL2 is available at configure time.

## Importing the package

```rust
const window = @import("window")
```

## Core lifecycle

```rust
const win window.Window =
  window.create("Demo", 800i64, 600i64)

window.clear(win)
window.present(win)
window.close(win)
```

The current API creates windows hidden by default. Visible manual demos call
`window.show(win)` explicitly.

## Events and input

```rust
const evt window.Event? = window.pollEvent(win)

if (evt != null) {
  print(window.eventKind(evt))
}
```

Current input/event helpers include:

- `pollEvent(win) -> Event?`
- `eventKind(evt) -> str`
- `eventKeyCode(evt) -> i64`
- `isKeyDown(win, keyCode) -> bool`
- `mouseX(win) -> i64`
- `mouseY(win) -> i64`
- `KEY_SPACE`
- `KEY_ESCAPE`

## Rendering helpers

The package already exposes a small immediate-mode drawing surface:

```rust
window.clearRgb(win, 32i64, 64i64, 96i64)
window.fillRect(win, 8i64, 8i64, 24i64, 24i64, 255i64, 0i64, 0i64)
window.drawText(win, 8i64, 40i64, "Score: 12", 2i64, 255i64, 255i64, 255i64)
window.present(win)
window.delay(1i64)
```

`drawText` uses a built-in bitmap monospace font, accepts printable ASCII,
supports `\n` for multi-line text, and clips against the window surface. The
`scale` argument must be at least `1`.

That is enough for the rectangle-rendered Flappy Bird demo with an in-window
score HUD in
`examples/flappy_bird.mog`.

## Build note

If SDL2 is not installed, the main interpreter still builds normally and simply
skips the optional `window` package target.

## Publishing the official package

The repo now carries a scripted publish path for the official package:

```bash
./scripts/publish_official_window.sh --registry-path /tmp/mog-registry
```

The script:

- checks that `build/packages/mog/window/package.so` or `package.dylib` exists
- uses the existing `mog publish` flow under the hood
- can also publish through an existing registry alias with
  `--registry <alias> --workspace <dir>`

If SDL2 was not available when the repo was built, the script fails early with
an explicit missing-artifact message instead of publishing a partial package.
